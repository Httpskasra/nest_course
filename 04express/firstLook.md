قبل از آموزش **Express.js** به بچه‌ها (با فرض اینکه سن و سطح مناسبی دارند و قبلاً **JavaScript** و **TypeScript** را یاد گرفته‌اند)، باید مفاهیم تئوری و عملی زیر را به‌طور کامل به آن‌ها یاد بدهید. Express یک فریم‌ورک **بک‌اند** برای Node.js است، بنابراین بدون دانستن اصول سمت سرور و HTTP، درک آن سخت می‌شود.

در ادامه **لیست پیش‌نیازهای تئوری** را به ترتیب آموزشی آورده‌ام:

---

## ۱. آشنایی با Node.js و اکوسیستم آن (بدون Express)
- **Node.js چیست؟**  
  - اجرای JavaScript خارج از مرورگر  
  - معماری رویداد محور (event-driven) و غیرهمزمان (non-blocking I/O)
- **مدیریت بسته‌ها با npm یا yarn**  
  - `npm init`، نصب پکیج‌ها، فایل `package.json`  
  - تفاوت `dependencies` و `devDependencies`
- **ماژول‌ها در Node.js**  
  - `require` و `module.exports` (CommonJS)  
  - آشنایی با `import/export` (ES modules) و نحوه فعال‌سازی آن  
- **فراخوانی توابع غیرهمزمان**  
  - Callback, Promise, async/await (که قبلاً در JS دیده‌اند ولی باید در محیط Node مرور شود)

## ۲. مفاهیم پایه شبکه و وب (HTTP)
- **مدل کلاینت-سرور**  
  - درخواست (Request) و پاسخ (Response)  
  - نقش مرورگر، API، سرور
- **پروتکل HTTP**  
  - متدهای رایج: GET, POST, PUT, DELETE, PATCH  
  - کدهای وضعیت (Status Codes):  
    - 200 OK، 201 Created، 400 Bad Request، 401 Unauthorized، 404 Not Found، 500 Internal Server Error
  - هدرها (Headers): `Content-Type`, `Authorization`, `Cookie`, …
  - بدنه (Body) در درخواست و پاسخ (فرمت JSON، فرم داده، متن ساده)
- **آشنایی با URL و پارامترها**  
  - Query string: `?name=ali&age=20`  
  - Path parameters: `/users/:id`
- **مفاهیم REST API** (ساده)  
  - منبع (Resource) و عملیات استاندارد روی آن با متدهای HTTP

## ۳. کار با ماژول `http` در Node.js (پیش از Express)
- ایجاد یک سرور ساده با `http.createServer`  
- خواندن درخواست (req.url, req.method, req.headers)  
- نوشتن پاسخ (res.writeHead, res.end, res.write)  
- چرا Express این فرآیند را ساده‌تر می‌کند؟ (دستکاری مستقیم با http خسته‌کننده است)

## ۴. مفاهیم Middleware (میان‌افزار) – تئوری
- تعریف: توابعی که به درخواست و پاسخ دسترسی دارند و تصمیم می‌گیرند ادامه دهند یا متوقف کنند  
- زنجیره میان‌افزارها (Request-Response cycle)  
- کاربردها: لاگ کردن، احراز هویت، پارس کردن بدنه، مدیریت خطا، فشرده‌سازی  
- تفاوت میان‌افزار با کنترلر (Handler نهایی)

## ۵. مفاهیم Routing
- نگاشت مسیر (route) + متد HTTP به یک تابع (کنترلر)  
- پارامترهای مسیر و کوئری  
- کار با چندین فایل route (مدولار کردن)

## ۶. کار با داده‌ها (قبل از دیتابیس)
- **دریافت داده از درخواست**  
  - Query string, params, body  
  - نحوه parse کردن JSON و form data (با `express.json()` و `express.urlencoded()`)
- **ارسال پاسخ JSON** (رایج در APIها)
- **سرویس فایل‌های ایستا** (مفهوم static files)

## ۷. مدیریت خطا در Express
- خطاهای همزمان (try/catch)  
- خطاهای غیرهمزمان در middleware (ارسال به next)  
- middleware مخصوص خطا (با ۴ پارامتر: err, req, res, next)

## ۸. آشنایی مقدماتی با محیط‌های توسعه و اجرا
- متغیرهای محیطی (environment variables) و فایل `.env`  
- پورت و host  
- ابزارهایی مانند `nodemon` برای restart خودکار

## 🧠 تئوری کامل قبل از تمرین Express (آنچه باید در کلاس یا آموزش توضیح دهید)

### **فصل ۱: سرور چیست و چرا به آن نیاز داریم؟**
- درک تفاوت وب‌سایت ایستا (HTML/CSS/JS) با وب‌اپ داینامیک  
- وقتی مرورگر به آدرس `http://localhost:3000/users` می‌رود، چه اتفاقی می‌افتد؟  
- نقش سرور در دریافت درخواست، پردازش (مثلاً از دیتابیس گرفتن) و برگرداندن پاسخ

### **فصل ۲: چرخه حیات یک درخواست HTTP در Express**
1. کلاینت درخواست می‌فرستد
2. Express درخواست را دریافت می‌کند
3. Request object ساخته می‌شود
4. میان‌افزارها به ترتیب اجرا می‌شوند
5. Route matching پیدا می‌شود
6. کنترلر اجرا می‌شود
7. Response object ساخته و ارسال می‌شود
8. اتصال بسته می‌شود (یا برای keep-alive باز می‌ماند)

### **فصل ۳: تفاوت Express با Node.js خالص**
- با Node.js خالص: باید خودتان مسیریابی، پارس بدنه، مدیریت فایل‌های ایستا، middlewareها را از صفر بنویسید  
- Express فقط یک لایه بالاتر از `http` است و این کارها را با کد کم و استاندارد انجام می‌دهد

### **فصل ۴: آشنایی با توابع کنترلر و next**
- هر middleware یک تابع است: `(req, res, next) => { ... }`  
- فراخوانی `next()` یعنی برو به middleware بعدی  
- اگر `next` صدا زده نشود و پاسخ ارسال نشود، درخواست می‌ماند (hangs)

### **فصل ۵: ساختار یک اپلیکیشن Express معمولی**
```
my-app/
├── src/
│   ├── routes/          # تعریف مسیرها
│   ├── controllers/     # منطق هر مسیر
│   ├── middlewares/     # میان‌افزارهای سفارشی
│   ├── models/          (بعداً با دیتابیس)
│   └── app.ts           # کانفیگ Express
├── public/              # فایل‌های ایستا (CSS, images)
├── .env
└── server.ts            # شروع سرور
```

  - `console.log` داخل middlewareها تا ترتیب اجرا را ببینند  
  - خطاهای عمدی ایجاد کنید تا middleware خطا کار کند

## ⚠️ نکات کلیدی برای تدریس به بچه‌ها
- **از تشبیه استفاده کنید**:  
  - سرور مثل یک فروشنده است، درخواست مثل سفارش، مسیر مثل آیتم منو  
  - Middleware مثل بازبینی‌کننده‌های زنجیره‌ای (کارت شناسایی، بسته‌بندی، ...)
- **قبل از Express حتماً یک سرور با `http` بنویسند** تا درک کنند که Express چه مشکلی را حل می‌کند  
- **ابتدا بدون TypeScript تمرین کنند** (تنها با JS) چون Express در TS نیاز به تایپ‌های اضافه (`@types/express`) دارد و ممکن است پیچیده شود  
- پس از چند مثال ساده در JS، همان کدها را با TS بازنویسی کنید تا مزیت type safety را ببینند

اگر این مفاهیم را کامل بفهمند، یادگیری Express برایشان بسیار روان و لذت‌بخش خواهد شد.
