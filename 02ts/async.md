کامل توضیح می‌دم که **Promise** چیه و چه فرقی با خروجی معمولی (غیر Promise) داره.

---

## Promise چیست؟ (مفهوم)

**Promise** در جاوااسکریپت/تایپ‌اسکریپت یک شیء (Object) است که **نتیجه یک عملیات ناهمگام (asynchronous)** را در آینده نشان می‌دهد.  
به‌عبارت ساده‌تر:  
- شما به یک قول (Promise) می‌دهید: «این کار انجام می‌شود، الان جوابش آماده نیست، اما به محض اینکه آماده شد به تو خبر می‌دهم.»
- تا آن زمان، خود Promise را دارید، نه مقدار نهایی را.

### ۳ حالت Promise
1. **Pending (در حال انتظار)**: کار هنوز تمام نشده.  
2. **Fulfilled (موفقیت‌آمیز)**: کار با موفقیت انجام شد و یک مقدار (value) تولید کرد.  
3. **Rejected (شکست‌خورده)**: کار با خطا مواجه شد و یک دلیل (error) برگرداند.

وقتی Promise به حالت Fulfilled یا Rejected برود، می‌گوییم **settled** شده است. فقط یک بار وضعیت عوض می‌شود.

## ساخت و مصرف Promise

### ساخت Promise
```typescript
const myPromise = new Promise<boolean>((resolve, reject) => {
  // عملیات ناهمگام...
  let success = true;
  if (success) {
    resolve(true); // fulfilled با مقدار true
  } else {
    reject("Something went wrong"); // rejected با یک خطا
  }
});
```
- `resolve` و `reject` دو تابع callback هستند که تعیین می‌کنند promise چطور تمام شود.

### مصرف (گرفتن نتیجه)
با `.then()` و `.catch()`:
```typescript
myPromise
  .then((result) => {
    console.log("موفقیت:", result); // true
  })
  .catch((error) => {
    console.error("خطا:", error); // "Something went wrong"
  });
```

همچنین می‌توان با `async/await` هم مصرف کرد:
```typescript
try {
  const result = await myPromise;
  console.log(result);
} catch (error) {
  console.error(error);
}
```

## چرا اصلاً Promise استفاده می‌کنیم؟

وقتی عملیات **زمان‌بر** است (درخواست شبکه، خواندن فایل، تایمر)، نمی‌خواهیم اجرای برنامه قفل شود.  
- **همگام (Synchronous)**: تا کار تمام نشود، کد بعدی اجرا نمی‌شود (مثل حلقه طولانی).  
- **ناهمگام (Asynchronous)**: کار شروع می‌شود، برنامه ادامه می‌دهد و وقتی کار تمام شد، نتیجه را دریافت می‌کنیم.

Promise مدلی مدرن برای مدیریت این ناهمگامی است (قبلاً فقط از callback استفاده می‌کردیم که به «جهنم callback» منجر می‌شد).

## تفاوت خروجی معمولی با Promise

حالا با مثال `sendEmail` تفاوت را نشان می‌دهم.

### حالت معمولی (همگام)
فرض کنید تابعی داشته باشیم که فوراً `boolean` برمی‌گرداند:
```typescript
function sendEmailSync(to: string, subject: string, body: string): boolean {
  // فرضاً ایمیل را همانجا می‌فرستد و جواب موفقیت را می‌دهد
  return true;
}

const result = sendEmailSync("a@b.com", "T", "B");
console.log(result); // true
console.log("ادامه برنامه...");
```
رفتار:
- `result` بلافاصله مقدار `true` است.
- می‌توانید مستقیماً با مقدار کار کنید (مقایسه، شرط، چاپ).
- تا تابع تمام نشود، خط بعدی اجرا نمی‌شود (blocking). اگر ارسال ایمیل واقعی ۳ ثانیه طول بکشد، برنامه ۳ ثانیه قفل می‌شود.

### حالت Promise (ناهمگام)
```typescript
function sendEmail(to: string, subject: string, body: string): Promise<boolean> {
  return new Promise((resolve) => {
    // شبیه‌سازی ارسال ایمیل با setTimeout (مثلاً ۲ ثانیه)
    setTimeout(() => {
      resolve(true);
    }, 2000);
  });
}

const result = sendEmail("a@b.com", "T", "B");
console.log(result); // Promise { <pending> } — نه true!
console.log("ادامه برنامه...");
```
اینجا:
- `result` یک **شیء Promise** است، نه `true`.
- خط «ادامه برنامه...» فوراً چاپ می‌شود، حتی اگر ارسال ایمیل هنوز تمام نشده باشد (non-blocking).
- برای گرفتن مقدار واقعی باید صبر کنید:
```typescript
result.then((value) => {
  console.log("نتیجه واقعی:", value); // true (بعد از ۲ ثانیه)
});
```

## جدول مقایسه

| ویژگی | خروجی معمولی (مثلاً `boolean`) | Promise (مثلاً `Promise<boolean>`) |
|-------|--------------------------------|-----------------------------------|
| **مقدار در لحظه** | بلافاصله مقدار واقعی | بلافاصله شیء Promise (نه مقدار) |
| **نوع بازگشتی** | `boolean` | `Promise<boolean>` |
| **دسترسی به مقدار** | مستقیماً (`result`) | با `.then()` یا `await` |
| **اجرای همزمان برنامه** | تا تمام نشود خط بعدی اجرا نمی‌شود (blocking) | کد بعدی بلافاصله اجرا می‌شود (non-blocking) |
| **مدیریت خطا** | `try/catch` معمولی | `.catch()` یا `try/catch` با `await` |
| **ترکیب عملیات** | کد پشت‌سرهم | زنجیره‌سازی `.then().then()` |
| **نمونه مناسب** | محاسبات سریع، دستکاری آرایه | درخواست HTTP، خواندن فایل، تایمر |

## چرا در `sendEmail` از Promise استفاده می‌کنیم؟

چون ارسال ایمیل واقعی یک **عملیات I/O** (شبکه) است و زمان می‌برد.  
اگر تابع به‌صورت همگام بود، سرور یا مرورگر تا پایان ارسال قفل می‌شد و پاسخگوی هیچ کار دیگری نبود.  
با برگرداندن `Promise<boolean>`، ما:
- بلافاصله کنترل را به برنامه برمی‌گردانیم.
- وقتی نتیجه آماده شد، از طریق `.then` یا `await` از موفقیت یا شکست آن مطلع می‌شویم.

همان‌طور که قبلاً اشاره کردم، `.then(console.log)` در مثال شما یک **callback** بود که به Promise وصل می‌شود تا بعداً صدا زده شود. این دقیقاً جایگزین مدرن callback‌های قدیمی است.

## جمع‌بندی

- **Promise** یک ظرف برای مقدار آینده است که وضعیتش (در انتظار، حل‌شده، رد‌شده) مشخص است.
- **تفاوت اصلی** با خروجی معمولی: خروجی معمولی «همین الان و مستقیم» در دسترس است، ولی Promise می‌گوید «صبر کن، خبرت می‌کنم».  
- در عمل، هر جا عملیات ناهمگام (زمان‌بر) داریم، از Promise استفاده می‌کنیم تا برنامه روان بماند و کد ما خواناتر شود.

اگر نیاز به مثال‌های بیشتر یا توضیح `async/await` داری، بگو تا برات باز کنم. 😊
